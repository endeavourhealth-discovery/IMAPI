## Step-by-Step Setup

### Step 1: Prerequisites Check (1 min)

```bash
# Check Java version
java -version
# Expected: OpenJDK Runtime Environment (build 21.x.x)

# Check Maven
mvn -version
# Expected: Apache Maven 3.6.0+
```

If these fail, install:
- Java 21: `$USERPROFILE\.jdks\corretto-21.0.8`
- Maven: Included in project gradlew

### Step 2: Copy Test File (1 min)

```bash
# From project root: Z:/IdeaProjects/Endeavour/InformationManager/IMAPI

# Create test directory if needed
mkdir -p src/test/java/org/endeavourhealth/imapi/transforms/compass

# Copy test file
cp AI-Specs/IMQtoSQL/5.Test.java src/test/java/org/endeavourhealth/imapi/transforms/compass/

# Verify
ls -la src/test/java/org/endeavourhealth/imapi/transforms/compass/
# Should show: 5.Test.java
```

### Step 3: Copy Phase 3 Implementation (1 min)

```bash
# Create implementation directory
mkdir -p src/main/java/org/endeavourhealth/imapi/transforms/compass

# Copy converter
cp AI-Specs/IMQtoSQL/4.Converter.java src/main/java/org/endeavourhealth/imapi/transforms/compass/

# Copy configuration to resources
mkdir -p src/main/resources
cp AI-Specs/IMQtoSQL/5.MappingConfiguration.yaml src/main/resources/
```

### Step 4: Run Tests (2 min)

```bash
# Full compilation and test
mvn clean test -Dtest=IMQtoSQLConverterTest

# Watch for output like:
# [INFO] Tests run: 21, Failures: 0, Errors: 0, Skipped: 6
# [INFO] BUILD SUCCESS
```

## Test Execution

### Option A: Unit Tests Only (Recommended - 3 sec)

```bash
# Run only unit tests (no database needed)
mvn test -Dtest=IMQtoSQLConverterTest -DfailIfNoTests=false

# You should see:
# âœ“ 1.1: Patient Query Validation
# âœ“ 1.2: Empty Query Validation
# âœ“ 1.3: Null Query Validation
# ... (18 more tests)
# âœ“ BUILD SUCCESS
```

### Option B: With Coverage Report (8 sec)

```bash
# Generate test coverage
mvn clean test jacoco:report

# View report
open target/site/jacoco/index.html
# Shows coverage percentage for each class
```

### Option C: Run Single Test

```bash
# Run one specific test
mvn test -Dtest=IMQtoSQLConverterTest#testSimplePatientQueryConversion

# Run tests matching pattern
mvn test -Dtest=IMQtoSQLConverterTest#*DateCondition*
```

### Option D: Integration Tests (with Database)

```bash
# Setup (requires PostgreSQL running with Compass schema)
export COMPASS_DB_AVAILABLE=true
export COMPASS_DB_URL="jdbc:postgresql://localhost:5432/compass"
export COMPASS_DB_USER="compass"
export COMPASS_DB_PASSWORD=""

# Run all tests including integration
mvn clean test
```

## Debugging Tips

### Problem: "Test class not found"

```
[ERROR] No tests were executed!
```

**Fix**: Check file path:
```bash
ls -la src/test/java/org/endeavourhealth/imapi/transforms/compass/5.Test.java

# If missing, copy again:
cp AI-Specs/IMQtoSQL/5.Test.java src/test/java/org/endeavourhealth/imapi/transforms/compass/
```

### Problem: "Converter class not found"

```
[ERROR] java.lang.ClassNotFoundException: IMQToSQLConverter
```

**Fix**: Check Phase 3 file:
```bash
ls -la src/main/java/org/endeavourhealth/imapi/transforms/compass/4.Converter.java

# If missing, copy it:
cp AI-Specs/IMQtoSQL/4.Converter.java src/main/java/org/endeavourhealth/imapi/transforms/compass/
```

### Problem: "Configuration not found"

```
[WARN] Mapping configuration not found in classpath
```

**Fix**: Copy configuration:
```bash
cp AI-Specs/IMQtoSQL/5.MappingConfiguration.yaml src/main/resources/
```

### Problem: "Compilation errors"

```
[ERROR] Compilation failure
[ERROR] ERROR [1,1]: package org.endeavourhealth.imapi.model.imq does not exist
```

**Fix**: Ensure project is properly built:
```bash
mvn clean install -DskipTests

# Then try tests again:
mvn test
```

### Show SQL Queries

```bash
# Run tests with verbose logging
mvn test -X 2>&1 | grep "Generated SQL"

# Example output:
# Generated SQL: SELECT * FROM patient WHERE organization_id = 'org-123'
# Generated SQL with date condition: SELECT * FROM patient WHERE organization_id = 'org-123' AND date_of_birth < '1960-01-01'
```

### Common Questions

**Q: Do I need a database?**  
A: No! Unit tests work without database. 15/21 tests pass immediately. âœ…

**Q: How long do tests take?**  
A: ~2-3 seconds for unit tests. 8-15 seconds with database (if available).

**Q: Can I run a single test?**  
A: Yes! `mvn test -Dtest=IMQtoSQLConverterTest#testSimplePatientQueryConversion`

**Q: Are integration tests required?**  
A: No. They're skipped automatically if database unavailable. Unit tests are sufficient for dev.

**Q: What about code coverage?**  
A: Run `mvn clean test jacoco:report` to generate detailed coverage report.
